构建流程 Build Flow

用户点击 "Build All" 后，系统的完整执行链路 — 从拓扑排序到并行 Agent 执行的全流程。

01 👤 用户 User 点击 "Build All"
用户在画布工具栏点击构建按钮，触发全量构建任务。
User clicks the Build All button on the canvas toolbar.
dispatch
02 🎨 画布 Canvas 发送节点与边至 SchemaEngine
Canvas 将当前图的 nodes[] + edges[] 传递给 SchemaEngine，准备序列化。
Canvas passes current graph nodes + edges to SchemaEngine for serialization.
serialize
03 ⚙️ SchemaEngine 序列化为 YAML
SchemaEngine 将图结构序列化为标准 YAML schema 文件，供 Agent 读取。
Graph structure is serialized into a YAML schema file consumed by the Agent runner.
topo input
04 🎨 画布 Canvas 发送节点与边至拓扑排序
同时将相同图数据发送给 TopoSort 模块，计算执行顺序。
Same graph data is forwarded to TopoSort to compute a dependency-ordered execution plan.
run algorithm
05 🔀 拓扑排序 TopoSort 运行 Kahn 算法
使用 Kahn's Algorithm 进行拓扑排序，将图划分为多个 Wave（执行批次），保证依赖顺序。
Kahn's BFS topological sort groups nodes into dependency waves (Wave 0, Wave 1 ...).
emit waves
06 🤖 AgentRunner 接收 Wave 0：[节点A, 节点B]
TopoSort 将 Wave 0（无依赖的节点批次）推送给 AgentRunner，包含节点A 和 节点B。
TopoSort emits Wave 0 — the first independent batch — to AgentRunner.
spawn parallel
Wave 0 · Parallel Execution
节点 A AgentRunner → AI CLI
AgentRunner 为节点A 独立生成提示词并调用 AI CLI，
AI CLI 以 SSE 形式实时流式返回输出。
SSE Stream 
节点 B AgentRunner → AI CLI
AgentRunner 为节点B 同步启动独立 AI CLI 进程，
两个进程并发执行，互不阻塞。
SSE Stream 
real-time status
08 🤖 AgentRunner 推送实时状态至画布
AgentRunner 将 SSE 流转发给 Canvas，画布上节点实时显示进度、日志与状态颜色。
AgentRunner relays streamed output to Canvas — nodes update color and log in real time.
wave 0 done → next
Wave 1 · Sequential
节点 C AgentRunner → AI CLI
节点A 和 节点B 均成功后，节点C（依赖前两者）进入队列。
AgentRunner 顺序执行 Wave 1，以相同方式调用 AI CLI 并流式回传结果。

Runs only after Wave 0 fully completes. Additional waves follow the same pattern.
all waves complete
