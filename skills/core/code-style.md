# Code Style

- Use descriptive names; avoid abbreviations unless universally known
- Functions do one thing; keep them under 40 lines
- No dead code, no commented-out blocks
- Prefer explicit over implicit; avoid magic numbers
- Consistent formatting: follow the project's existing style
